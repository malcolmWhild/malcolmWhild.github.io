Data Processing

Coming from a pre-press/graphic design background, databases and associated technologies has been an exciting step in a new direction for me. I find that it appeals to the logical side of my nature, and it is very rewarding to program queries that will clean and process large amounts of data very quickly.

When any of the elements of a DM campaign are to be mailed, the first step is to clean the data so that it complies with NZ Post standards. I import supplied data files into Access where I merge then if necessary and add standard DM fields. I populate the DM fields using using SQL queries which I order and save in case the data is resupplied for any reason. The queries perform tasks such as splitting addresses on commas, checking/flagging overseas records, extracting postcodes, de-duping, processing for SOA, mailcasing, flagging GNA records, splitting out into runs/batches and postcode sorting. Access is a good tool, but for large databases I would prefer to use the power of SQL Server. With this in mind, I'm working towards using C# to create a data processing application which will interface directly with SQL Server and guide the user through the steps necessary to import and standardise data.
