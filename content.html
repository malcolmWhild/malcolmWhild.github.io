Data Processing

Because I come from a pre-press/graphic design background, databases and associated technologies have been an exciting step in a new direction for me. I find that it appeals to the logical side of my nature, and it is very rewarding to program queries that will quickly process large amounts of data.

When any of the elements of a DM campaign are to be mailed, the first step is to clean the data so that it complies with NZ Post standards. Data processing is done in Access using SQL queries which perform tasks such as checking/flagging overseas records, extracting postcodes, de-duping, processing for SOA, mailcasing, flagging GNA records, splitting out into runs/batches and postcode sorting.

Indesign and VDP Document Setup

I have many years' experience using Indesign and I am enthusiastic about producing functional, well-organised and documented print-ready files. I’m used to dealing with supplied files and with templates, so I’m able to quickly understand the structure of existing documents.

Supplied Indesign files are checked to ensure that there is sufficient bleed, that fonts and images are properly supplied and that all images are checked and resized if necessary to optimise performance of generated PDFs through the RIP.

The processed data is used to create a plan file and any campaign logic is added in uPlan – these might be ADORs for text placeholders, layer visibility, and barcodes. QLingo logic is used extensively, while Javascript is used for loops and for managing arrays.

Multi-channel Campaigns

Multi-channel campaigns are the part of my work that I look forward to most. I work on campaigns ranging from simple sign-up forms, which create new records as they capture recipient data, to microsites which target groups of existing recipients. All web pages and emails are fully responsive, and logic is used to personalise the design to target specific groups.

It is very rewarding to be able to create a smooth-running multi-channel campaign that looks great on any device, and to be able to track recipient engagement in real time. This type of work is a very satisfying addition to my pre-press experience.

With methods of communication constantly evolving, it is exciting to be at the forefront of developments in utilising clients’ data to effectively target their customer base.

If there is a print component to the campaign, data is processed and sorted to NZ Post requirements in the same way as for a standalone print campaign. The records are then split into post and email batches and any campaign-specific fields are added. The working table is then imported directly into SQL Server and linked to a web-ready plan file.

Once the plan and data source are uploaded to the uProduce server, web pages and emails are created in Visual Studio Code. Emails are inlined and imported to Dreamweaver for ADORs to be placed.

Campaign reports are created using the uProduce Marketing Console and are linked to a Circle plan so that the client can track email opens, page visits and completion rates along with other valuable information to help them determine the success of their campaign.
